## What the script does

1. **Loads credentials** — reads `LIVEKIT_URL`, `LIVEKIT_API_KEY`, and `LIVEKIT_API_SECRET` from `.env.local` via `python-dotenv`. Raises a clear `EnvironmentError` if any value is missing.

2. **Lists active rooms** — opens an async `LiveKitAPI` context manager and calls `lk.room.list_rooms(ListRoomsRequest())`. If no rooms are active it logs that fact and exits cleanly.

3. **Broadcasts to every room** — for each room it calls `lk.room.send_data()` with a `SendDataRequest` containing:
   - `room` — the room name
   - `data` — the UTF-8-encoded system message
   - `kind=0` — `RELIABLE` delivery (guaranteed, ordered)

4. **Logs everything** — a `logging` setup writes timestamped lines to both stdout and `/home/user/livekit-broadcast/broadcast.log`. Per-room success/failure and a final summary are recorded.

## Key design decisions

| Detail | Choice |
|---|---|
| `kind` value | `0` (RELIABLE) — confirmed from the SDK's protobuf enum (`RELIABLE=0, LOSSY=1`) |
| `LiveKitAPI` usage | Used as an async context manager (`async with`) to ensure the underlying HTTP session is always closed |
| Error isolation | Each room's `send_data` call is wrapped individually so one failure doesn't abort the rest |
| Credential source | `.env.local` via `load_dotenv`; env vars can also override at process level |
