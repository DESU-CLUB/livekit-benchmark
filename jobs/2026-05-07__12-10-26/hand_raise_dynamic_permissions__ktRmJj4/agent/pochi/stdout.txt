### What the server does

| Endpoint | Method | Behaviour |
|---|---|---|
| `/hand-raise` | POST | Toggles `canPublish` for the given participant and calls `RoomServiceClient.updateParticipant` |
| `/hands-up` | GET | Returns a JSON array of identities that currently have `canPublish: true` |

### Key implementation details

- **Credentials** — `LIVEKIT_URL`, `LIVEKIT_API_KEY`, and `LIVEKIT_API_SECRET` are read exclusively from environment variables; nothing is hardcoded.
- **In-memory state** — a `Map` (`handRaiseState`) tracks each identity's current `canPublish` value. Absent entries default to `false`.
- **Toggle logic** — `newCanPublish = !currentState`; the Map is updated only _after_ `updateParticipant` resolves.
- **Validation** — missing `room` or `identity` returns HTTP 400 immediately.
- **Permissions sent to LiveKit** — `{ canPublish: newCanPublish, canSubscribe: true }`.
