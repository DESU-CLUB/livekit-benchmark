I have implemented the Node.js token server per your requirements.

The server uses Express and the `livekit-server-sdk` to issue and refresh tokens. It listens on port 3000 and exposes the following endpoints:

1. `GET /health`
   Returns a `200 OK` status to indicate the server is running.

2. `GET /token`
   Issues a short-lived token with a 30-minute TTL (`ttl: '30m'`). You can pass the room name and participant identity via query parameters (e.g., `?room=myroom&identity=myuser`). It defaults to random values if not provided.

3. `POST /refresh`
   Allows clients to renew their tokens before expiry. It expects the old token either in the `Authorization: Bearer <token>` header or in the JSON body as `{ "token": "<token>" }`. It verifies the old token and issues a new one with the exact same claims (identity, room, etc.) but a fresh 30-minute TTL.
